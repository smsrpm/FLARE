# **Preface**

FLARE (Fast Licensing Accident Response Engine) is a single-node, point-kinetics-based reactor safety analysis code developed for the rapid evaluation of design-basis transients and accidents in pressurized water reactors (PWRs) and small modular reactors (SMRs). The code is implemented as a single Python module (flare_sim.py) and is designed for use in both interactive (Streamlit) and batch computing environments.

This Code Theory Manual documents the mathematical models, constitutive relations, and numerical methods that underlie the FLARE simulation engine. The document is organized to parallel the structure of standard nuclear safety analysis code documentation, beginning with the regulatory context and scope, proceeding through the governing equations and closure models for each physical domain, and concluding with the numerical solution strategy and code architecture.

The development of FLARE draws on the same phenomenological foundations as established industry codes such as RELAP5-3D, TRACE, and CATHARE. Where FLARE departs from those codes вҖ” principally in its use of a single-node, homogeneous equilibrium mixture approximation for the bulk coolant вҖ” the basis and limitations of that departure are explicitly described. The code is intended for scoping calculations, sensitivity studies, and Monte Carlo uncertainty analysis where computational throughput is of primary importance.

# **Chapter 1 вҖ” Introduction and Regulatory Context**

## **1.1  Purpose and Scope**

Design-basis accident (DBA) analysis in the United States is governed by Title 10 of the Code of Federal Regulations (10 CFR), Parts 50 and 52, and supported by Regulatory Guide 1.203, Transient and Accident Analysis Methods [NRC, 2005]. For each postulated accident, the licensee must demonstrate that the principal fission-product barriers вҖ” the fuel cladding, the reactor coolant system (RCS) pressure boundary, and the containment вҖ” remain intact or fail within prescribed limits. The quantitative acceptance criteria most relevant to FLARE applications are specified in 10 CFR 50.46 and are summarized in Table 1-1.

| **Criterion** | Limit | FLARE Output Variable |
| --- | --- | --- |
| Peak Cladding Temperature (PCT) | 1204В°C (2200В°F) | T_hot_clad_arr |
| Maximum Local Oxidation (MLO) | 17% of pre-accident cladding thickness | Computed from Baker-Just integral |
| Core-wide oxidation | 1% of total cladding reacting with water | N_pins Г— MLO fraction |
| Coolable core geometry | No criteria exceeded | Implicit in PCT < 1204В°C |
| Long-term cooling | Core can be cooled indefinitely | RCS level, ECCS flow |

FLARE is designed to support the following transient and accident categories:

- Large-Break Loss-of-Coolant Accident (LBLOCA) вҖ” double-ended guillotine breaks in the cold leg or hot leg

- Small-Break Loss-of-Coolant Accident (SBLOCA) вҖ” spectrum of break sizes up to 10-inch equivalent diameter

- Reactivity-Initiated Accidents (RIA) вҖ” rod ejection and rod withdrawal at power and at zero power

- Anticipated Transients Without Scram (ATWS) вҖ” loss of feedwater, loss of flow, with failed reactor trip

- Steam Generator Tube Rupture (SGTR) and Main Steam Line Break (MSLB)

- Loss of Forced Circulation and transition to natural circulation

## **1.2  Code Architecture**

FLARE is organized as follows. The simulation engine is contained in a single Python module, flare_sim.py, which accepts an Excel-format input file and produces Excel and CSV output. The Streamlit-based user interface (flare_ui.py) provides interactive case setup, parameter overrides, simulation execution, and graphical post-processing. The uncertainty analysis module (flare_ua.py) wraps the simulation engine in a Monte Carlo sampling loop to support Best-Estimate Plus Uncertainty (BEPU) applications consistent with the statistical methodology of 10 CFR 50.46(b).

The core simulation function is pwr_sim(), which reads the input command block, initializes all state variables, and advances the solution forward in time using an operator-split strategy. Post-processing is performed by the post_processing() function, which generates plots and writes the output workbook.

## **1.3  Single-Node Approximation**

FLARE applies a lumped-parameter, single-node (0-D) representation of the primary system. The entire reactor coolant system inventory вҖ” vessel, hot leg, cold leg, pressurizer steam space, and loop volumes вҖ” is represented as a single well-mixed control volume with uniform pressure and specific enthalpy. This is the same homogeneous equilibrium model (HEM) used in the original FLASH code [Margolis and Redfield, 1966] and remains the basis of the FLASH modeling heritage that informs FLARE.

The single-node approximation is justified for the phenomena of primary interest in FLARE applications: system pressure history, bulk coolant enthalpy, break mass flow rate, and integrated heat input to the coolant. For spatial distributions of temperature and void fraction within the reactor vessel вҖ” which require multi-node or sub-channel treatment вҖ” FLARE relies on the hot-channel model described in Chapter 4.

# **Chapter 2 вҖ” Thermal-Hydraulic System Model**

The thermal-hydraulic model in FLARE is based on the homogeneous equilibrium mixture (HEM) approximation, in which the liquid and vapor phases of the coolant are assumed to move together at the same velocity and to be in local thermodynamic equilibrium. This reduces the two-fluid conservation equations to a single set of mixture equations. The governing equations are closed by the IAPWS-IF97 steam tables, implemented via the XSteamPython library.

## **2.1  Conservation of Mass**

The global mass balance for the RCS control volume is:

dMdt=minвҲ’mbreakвҲ’mPORVвҖҒ2.1

where M is the total fluid mass [kg] in the control volume, m_in is the mass flow rate from ECCS injection sources (primarily the accumulator) [kg/s], m_break is the break mass flow rate [kg/s], and m_PORV is the PORV relief valve mass flow rate [kg/s]. This equation is integrated explicitly in time:

Mt+О”t=Mt+minвҲ’mbreakвҲ’mPORVО”tвҖҒ2.2

The specific volume of the mixture is determined from the equation of state at each time step, closing the volume conservation equation:

Vsys=MвҖүvP,hвҖҒ2.3

where V_sys is the total system volume [mВі], held constant throughout the transient, and v(P,h) is the specific volume [mВі/kg] from the steam tables at the current pressure P and mixture specific enthalpy h.

## **2.2  Conservation of Energy**

The mixture energy equation, written in terms of specific enthalpy, is derived from the first law of thermodynamics for an open control volume. In the formulation adopted by FLARE, following the FLASH code tradition [Margolis and Redfield, 1966], the conservation equations are cast as a coupled 2Г—2 linear system in the pressure and enthalpy increments. The right-hand side (b vector) of that system contains the net energy and volume source terms:

b0=minhinвҲ’eвҲ’mbreakhbвҲ’eвҲ’mPORVhpвҲ’e+QnetвҲ’QSGвҖҒ2.4

b1=вҲ’vminвҲ’mbreakвҲ’mPORVвҖҒ2.5

where e is the specific internal energy [J/kg], h_in is the enthalpy of the injected water [J/kg], h_b is the stagnation enthalpy at the break [J/kg], h_p is the PORV discharge enthalpy [J/kg], Q_net is the net thermal power delivered to the coolant [W], and Q_SG is the steam generator heat removal rate [W].

The 2Г—2 coefficient matrix A is formed from the partial derivatives of specific internal energy and specific volume with respect to pressure and enthalpy:

A=MвҲӮeвҲӮPвҲӮeвҲӮhвҲӮvвҲӮPвҲӮvвҲӮhвҖҒ2.6

The pressure and enthalpy at the next time step are obtained from the solution of the linear system:

Pt+О”tht+О”t=Ptht+О”tвҖүAвҲ’1bвҖҒ2.7

The temperature T(t+О”t) is then recovered from the steam tables as T = f(P, h). This implicit formulation provides numerical stability without iteration, following the semi-implicit method described by Liles and Reed [1978].

## **2.3  Equation of State and Fluid Properties**

All thermodynamic and transport properties of water and steam are evaluated using the IAPWS-IF97 formulation as implemented in XSteamPython. The following properties are evaluated at each time step:

- Saturation temperature T_sat(P) and saturation pressure P_sat(T)

- Saturated liquid and vapor specific volume, enthalpy, entropy, and density

- Subcooled liquid specific volume v(P,h) and temperature T(P,h)

- Mixture quality x_eq from equilibrium flash at (P,h)

- Specific heat cp, thermal conductivity k, and dynamic viscosity Ој for liquid and vapor

The equilibrium quality is determined at each step and drives the selection of critical flow models, break stagnation state, and heat transfer regime:

xeq=hвҲ’hfhfg,вҖҒвҖҒhfвүӨhвүӨhgвҖҒ2.8

Subcooled conditions (x_eq < 0) and superheated conditions (x_eq > 1) are handled as limiting cases.

## **2.4  Break Flow Model**

The break mass flow rate is one of the most important quantities in LOCA analysis, controlling the rate of inventory loss from the RCS. FLARE implements three models for critical (choked) flow at the break plane, selected automatically based on the thermodynamic state of the stagnation fluid at the break.

### **2.4.1  Equilibrium Rate Model (ERM)**

The Equilibrium Rate Model (ERM), based on Moody [1965], is the primary critical flow model for LOCA blowdown. It assumes that the fluid at the break throat is in thermodynamic equilibrium and that flashing is complete at the throat. The critical mass flux is obtained by maximizing the mass flux G with respect to the throat pressure P_c:

G=ПҒ3вҖү2hoвҲ’hcвҖҒ2.9

where the mixture density ПҒ at the throat depends on the slip ratio S between phases. FLARE supports three slip models: Homogeneous Equilibrium (S=1), Moody slip (S = (v_g/v_f)^(1/3)), and Fauske slip (S = (v_g/v_f)^(1/2)). The Fauske slip model is the default for LOCA applications.

The throat quality x_c is related to the stagnation quality x_o through isentropic flashing:

xc=xosfg,osfg,c+sf,oвҲ’sf,csfg,cвҖҒ2.10

The critical throat pressure P_c is found by Newton-Raphson iteration to locate the maximum of G(P_c). The ERM is valid for stagnation pressures representative of PWR full-power operating conditions (~15-16 MPa) where h_fg is large. Near the thermodynamic critical point, h_fg collapses and the ERM becomes non-physical. FLARE blends the ERM toward a subcooled Bernoulli model above approximately 16.4 MPa:

Gblend=1вҲ’wGERM+wGBernoulliвҖҒ2.11

where the blending weight w increases linearly from 0 at h_fg = 900 kJ/kg to 1 at h_fg = 150 kJ/kg. The Bernoulli subcooled orifice model is:

GBernoulli=2ПҒPвҲ’PcontainmentвҖҒ2.12

### **2.4.2  Henry-Fauske Subcooled Critical Flow**

For subcooled stagnation conditions (x_eq < 0), FLARE uses the ERM blended model with the subcooled liquid density evaluated from the steam tables. This provides a continuous transition from subcooled to two-phase flow as the bulk coolant inventory depletes and the stagnation state approaches saturation.

### **2.4.3  Break Discharge Coefficient**

The actual break mass flow rate is:

mbreak=CdAbreakGcriticalвҖҒ2.13

where A_break = ПҖ d_breakВІ / 4 is the break orifice area [mВІ] and C_d is the discharge coefficient. FLARE uses separate discharge coefficients for subcooled (C_d,sub) and two-phase (C_d,sat) conditions, reflecting the different vena contracta behavior of subcooled and flashing flows. Default values are C_d = 0.9 for both, consistent with the range 0.84вҖ“0.95 recommended in NRC validation studies.

### **2.4.4  Break Stagnation State Transition**

As RCS inventory depletes, the stagnation state at the break transitions through three tiers based on the normalised mass scaled parameter M_scaled = M(t)/M(t=0) and the equilibrium steam quality x_eq. The selection logic is applied independently in both the subcooled (x_eq < 0) and two-phase (x_eq вүҘ 0) branches:

вҖў  M_scaled > 1.0: The RCS is overfilled (ECCS has added more than the initial inventory). The break is submerged in cold injected liquid; h_stag = h_L(P), the saturated liquid enthalpy at the current pressure.

вҖў  x_eq < 0.01 (trace quality, essentially liquid-filled core): h_stag = h_L(P). This tier correctly captures the late reflood phase when ECCS water occupies most of the vessel but a tiny steam fraction keeps x_eq marginally positive.

вҖў  All other conditions: a latching flag (flag_stagnation_break) selects between the mixture enthalpy h_mix (when M_scaled > transition_mixture_mass, default 0.41) and saturated vapor h_g (when M_scaled вүӨ 0.41, representing steam venting from a largely voided vessel). The vapor latch resets when M_scaled rises back above the threshold and x_eq drops below 0.01, preventing historically latched steam enthalpy from persisting incorrectly into the reflood phase.

Dynamic choking is evaluated at each time step. When the ratio P_containment / P_RCS falls below the critical pressure ratio (default 0.55), the flow is choked and the ERM is applied. When this ratio is exceeded (near-atmospheric RCS pressure), the flow transitions to a subcritical orifice model: G = C_d вҲҡ(2 ПҒ О”P), which correctly drives break flow to zero as P_RCS approaches P_containment. The break stagnation enthalpy is recorded in the output as the Break Enthalpy (kJ/kg) column.

## **2.5  PORV and Safety Relief Valve Model**

The Pilot-Operated Relief Valve (PORV) opens when the RCS pressure exceeds its setpoint P_PORV. The relief mass flow rate is calculated using the same ERM critical flow model as the break:

mPORV=Cd,PORVAPORVGERMP,hstagвҖҒ2.14

where h_stag is the stagnation enthalpy at the PORV connection point. Because the PORV is physically located at the top of the pressurizer, h_stag is selected based on the normalised pressurizer liquid level О¶ = pzr_level / pzr_height rather than the vessel level. When О¶ > transition_mixture_mass (default 0.41), the pressurizer contains significant liquid and h_stag = h_mix (the bulk mixture enthalpy), reflecting liquid-phase relief. When О¶ вүӨ 0.41, the pressurizer is mostly empty or steam-space dominant and h_stag = h_g (saturated vapour), reflecting steam venting. This formulation correctly captures the transition to a water-solid condition (pressurizer empty, RCS overfilled) where the PORV relieves subcooled liquid rather than steam.

A predictor-corrector method is used to determine the PORV flow fraction that pins the pressure at its setpoint. An under-relaxation factor P_relax is applied to the pressure update to damp numerical oscillations near the setpoint:

Pt+1=1вҲ’PrelaxPt+PrelaxPcomputedвҖҒ2.15

## **2.6  Pressurizer Model**

The pressurizer is modeled as a lumped-parameter surge volume connected to the RCS via the surge line. When the pressurizer has liquid (level between 0 and pzr_level_hi), it maintains the RCS at the rated operating pressure P_operating by absorbing volume changes through surge line flow. The surge line mass flow rate is back-calculated from the volume conservation equation:

msurge=О”VvfО”tвҖҒ2.16

where О”V is the change in RCS specific volume multiplied by total mass. During blowdown (when a break is open), the pressure hold is disabled and the pressurizer drains via two mechanisms. In the PORV-open branch, liquid in the pressurizer steam space flashes to replace the steam vented through the PORV; the pressurizer level drops at a rate proportional to the PORV steam flow divided by the liquid density, and an insurge of RCS liquid flows up the surge line to thermally support the flashing. In the steam-expansion branch (no PORV flow), the steam bubble expands as pressure falls, pushing liquid down the surge line into the RCS as an outsurge. When the pressurizer level reaches zero (empty pressurizer, water-solid RCS condition), surge flow is set to zero in both branches вҖ” there is no liquid remaining to flash or to transfer via the surge line. The pressurizer model supports startup, ATWS, and LOHS transient modeling where the pressurizer plays a primary pressure-control role.

## **2.7  Accumulator Injection Model**

Each accumulator is represented as a gas-pressurized water tank. Nitrogen gas occupies the upper portion of the tank and expands polytropically as the water level drops during injection. The gas pressure and temperature evolve as:

Pgast=PgastвҲ’1вҲ’baaaeвҲ’aaО”t+baaaвҖҒ2.17

where a_a accounts for the polytropic expansion rate and b_a accounts for heat transfer from the tank walls to the nitrogen. Injection flow begins when the accumulator gas pressure (accounting for the liquid head above the injection nozzle) exceeds the RCS pressure:

macc=Anozzle2Pgas+ПҒwgzвҲ’PRCSПҒw,вҖҒвҖҒPgas+ПҒwgz>PRCSвҖҒ2.18

The accumulator liquid volume, gas volume, gas pressure, and gas temperature are tracked at each time step. Default geometry represents a typical large PWR accumulator: 66.63 ftВІ cross-section, 10.5 ft length, with approximately 630 ftВі of initial water inventory.

### **2.7.2  ECCS Boron Injection Model**

Both the accumulator and the pumped SI sources (HPSI and LPSI) inject borated water into the RCS. The accumulator boron concentration is specified by acc_boron_ppm [ppm] and the SI boron concentration by si_boron_ppm [ppm]; these are independent parameters to allow modelling of boron dilution scenarios (e.g. si_boron_ppm = 0 for unborated SI). Default values are 2300 ppm for both, representative of a typical PWR RWST. The RCS boron mass is tracked by a first-order mass balance at each time step:

*d(m_B)/dt = mМҮ_acc Г— C_acc + mМҮ_SI Г— C_SI вҲ’ mМҮ_break Г— C_RCS*

where m_B is the total boron mass in the RCS [kg], C_acc and C_SI are the accumulator and SI boron concentrations [mass fraction], and C_RCS = m_B / M_total is the instantaneous RCS boron concentration. The RCS boron concentration in ppm is output as the RCS Boron (ppm) column. The reactivity worth of the injected boron is:

*ПҒ_boron = Оұ_boron [pcm/ppm] Г— (C_RCS [ppm] вҲ’ C_init [ppm])*

where Оұ_boron = alpha_boron_pcm_ppm (default вҲ’10 pcm/ppm) is the differential boron worth and C_init = rcs_boron_ppm_init (default 0 ppm, representing EOC conditions) is the initial RCS boron concentration. The boron reactivity is included in ПҒ_net at every time step and is also added to the SLCS reactivity in the Reactivity Boron (pcm) output column.

### **2.7.3  SI Pump Degradation Model**

As the reactor vessel refills during ECCS injection, the static back-pressure on the SI pumps increases, reducing delivered flow. FLARE models this effect through a vessel fill-fraction degradation factor applied to both HPSI and LPSI flow rates. The volumetric fill fraction is defined as:

*f_fill = ves_ll(t) / (R5_lowCV_height + R5_highCV_height)*

The onset of degradation is controlled by the user input si_fill_threshold (default 0.5). The degradation factor is unity for f_fill вүӨ si_fill_threshold and decreases linearly to zero at f_fill = 1.0 (pump fully shut off when the vessel is completely filled):

*f_SI = max(0, 1 вҲ’ max(0, f_fill вҲ’ si_fill_threshold) / max(1 вҲ’ si_fill_threshold, Оө))*

This degradation model prevents unbounded inventory accumulation during reflood and produces a smooth, physically motivated reduction in SI flow rather than the abrupt cutoff that would result from the si_mass_ceiling hard limit alone. The hard cutoff (si_mass_ceiling, default 1.60) is retained as a backstop. Setting si_fill_threshold = 1.0 disables degradation entirely, which is appropriate for intact-RCS scenarios such as boron dilution where column E (cvcs_mdot) represents normal-operation CVCS makeup/letdown flow rather than emergency injection.

## **2.7.4  Chemical and Volume Control System (CVCS) Makeup/Letdown Model**

The CVCS makeup/letdown model represents the chemical and volume control system that continuously circulates RCS coolant through a letdown path and returns it via a makeup path at a controlled boron concentration. The model is activated by setting cvcs_kgs [kg/s] to a nonzero value; it runs from cvcs_start_time_s (default 0 s) onwards. The net mass effect on the RCS is zero вҖ” makeup and letdown flows are equal and opposite. The net boron effect per timestep is:

*d(m_B)/dt|_CVCS = cvcs_kgs Г— (cvcs_boron_ppm вҲ’ C_RCS) Г— 10вҒ»вҒ¶*

where cvcs_boron_ppm is the boron concentration of the makeup water [ppm] (default 0, representing unborated makeup вҖ” the worst-case dilution scenario) and C_RCS is the instantaneous RCS boron concentration [ppm]. Setting cvcs_boron_ppm = C_RCS_init produces a neutral baseline with no dilution or boration effect. Column E of the time-series input table (cvcs_mdot [kg/s]) acts as an imbalance adder on top of the baseline: positive values represent extra makeup injection at cvcs_boron_ppm (adding mass and boron to the RCS), while negative values represent extra letdown (removing mass and boron at the current RCS concentration). For a boron dilution design-basis event, the recommended input settings are: cvcs_kgs = 5 (typical CVCS flow rate), cvcs_boron_ppm = 0 (unborated makeup), cvcs_start_time_s = time of dilution initiation, si_fill_threshold = 1.0 (to disable ECCS pump degradation in the intact-RCS condition), and ria_flag = 1 with appropriate kinetics parameters.

## **2.8  Steam Generator Heat Removal**

The steam generator (SG) heat removal rate Q_SG can be specified in one of two ways, selected by the sg_table_flag input parameter.

### **2.8.1  UA Model (sg_table_flag = 0)**

When the pump is operating (sg_flag = 1, sg_table_flag = 0), the SG heat removal is calculated from the overall heat transfer coefficient-area product UA_SG and the log-mean temperature difference between the primary and secondary fluids:

QSG=UASGTprimaryвҲ’TsecondaryвҖҒ2.19

where UA_SG is the overall heat transfer coefficient-area product [W/K] and T_secondary is the saturation temperature of the secondary steam at P_sec. The UA model is scaled with pump mass flow rate to account for the reduction in primary-side convective resistance as flow decreases during coastdown. The Q_SG is bounded above by the maximum heat that the primary flow can surrender to the secondary:

QSGвүӨmpumphprimaryвҲ’hsecondary,satвҖҒ2.20

### **2.8.2  Table-Driven Model (sg_table_flag = 1)**

When sg_table_flag = 1, column D of the time-series input table is used as a prescribed Q_SG boundary condition [MW vs time], evaluated by linear interpolation at each time step. Positive values indicate normal heat removal (primary to secondary). Negative values indicate heat addition to the primary, representing overcooling events such as a main steam line break (MSLB). The interpolated value is scaled by the fractional RCP mass flow rate relative to rated flow (pump_mdot(t) / pump_mdot_rated), reflecting the reduction in primary-side convective resistance as the pump coasts down. This scaling gives the correct limiting behaviour: Q_SG вҶ’ 0 as the pump trips, consistent with natural circulation transition.

## **2.9  Reactor Coolant Pump Model**

The RCP coastdown is modeled using a first-order exponential decay in pump speed:

Пүt=ПүratedeвҲ’t/П„pumpвҖҒ2.21

where Пү_rated is the rated pump speed [rpm] and П„_pump is the coastdown time constant [s] (typical PWR: 25вҖ“40 s). The coastdown begins at t_break (for LOCA cases) or at the time of the pump trip signal. When pump_scram_trip = 1, the pump is held at rated speed until the reactor scram fires, then coasts.

The pump mass flow rate, velocity, and hydraulic head at each coastdown speed are determined from four-quadrant homologous pump curves based on the Westinghouse model, implemented in the pump_module. The operating point on the homologous curves is determined by solving for the intersection of the pump head curve with the system resistance curve, represented by the orifice area parameter.

The core coolant velocity is derived from the pump mass flow rate:

vcore=mpumpПҒcoreAflow,coreвҖҒ2.22

where A_flow_core is the total coolant flow area through the core [mВІ], calculated from the pin geometry as A_flow_core = N_pins Г— (pitchВІ - ПҖ d_pinВІ / 4).

# **Chapter 3 вҖ” Core Heat Transfer Model**

The core heat transfer model calculates the cladding surface temperature, fuel centerline temperature, heat transfer coefficient, and critical heat flux at each time step. The model applies to the average pin in the core, with a hot-channel correction applied to estimate the peak (hot-pin) temperatures relevant to the 10 CFR 50.46 acceptance criteria.

## **3.1  Fuel Rod Geometry**

The default FLARE fuel rod geometry is based on the mPower SMR, with 18,200 Zircaloy-4 clad fuel pins in a square pitch array. Key geometric parameters are summarized in Table 3-1.

## **3.2  Cladding Temperature вҖ” Slab Heat Conduction**

The cladding is modeled as a 1-D planar slab with N computational nodes across its thickness Оҙ_clad. The transient heat conduction equation is discretized using an implicit finite-difference scheme:

ПҒcladcp,cladвҲӮTвҲӮt=О»cladвҲӮ2TвҲӮx2вҖҒ3.1

Boundary conditions: at the inner cladding surface (x = 0), the heat flux from fission and decay heat Q_total/SurfArea [W/mВІ] is applied. At the outer cladding surface (x = Оҙ_clad), convective heat transfer to the coolant is specified:

вҲ’О»claddTdxouter=ОұTouterвҲ’TcoolantвҖҒ3.2

where Оұ is the coolant-side heat transfer coefficient [W/mВІK]. The implicit Crank-Nicolson discretization produces a tridiagonal system solved at each time step. The solution yields the cladding outer surface temperature TTwall[t] вҖ” the primary figure of merit for cladding integrity assessment.

## **3.3  Fuel Temperature Model**

The fuel centerline and average temperatures are calculated from the quasi-steady-state thermal resistance chain, corrected for fuel thermal inertia using a first-order exponential lag:

### **3.3.1  Thermal Resistance Chain (Average Pin)**

The temperature rise through the average pin is decomposed into four resistances in series:

О”Tclad=qavgвҖІвҖІОҙcladО»cladвҖҒ3.3

О”Tfuel=Qpin4ПҖkfuelLheatedвҖҒвҖҒ[centerline]вҖҒ3.6

where q"_avg = Q_total/SurfArea is the average surface heat flux [W/mВІ], Q_pin = Q_total/N_pins is the average pin power [W], h_gap is the fuel-clad gap conductance [W/mВІK], and k_fuel is the UOвӮӮ thermal conductivity [W/mK].

### **3.3.2  Fuel Thermal Inertia**

The quasi-steady fuel temperature is approached with a first-order lag characterized by the fuel thermal time constant:

П„fuel=ПҒfuelcp,fuelrfuel24kfuelвҖҒ3.7

The actual fuel temperature at each step is computed as:

Tfuelt=Tfuel,sst+TfueltвҲ’1вҲ’Tfuel,ssteвҲ’О”t/П„fuelвҖҒ3.8

This lag captures the stored energy in the fuel pellet, which is important for the correct timing of Doppler feedback during power transients and for the adiabatic temperature rise in RIA events.

## **3.4  Hot-Channel Model**

The hot-pin cladding and fuel temperatures are estimated by applying the combined peaking factor F = F_r Г— F_z to the average pin quantities:

Thot clad,ss=TTwall+FвҲ’1qavgвҖІвҖІОұвҖҒ3.9

where F_r is the radial (inter-pin) peaking factor and F_z is the axial peaking factor (ratio of peak to average axial heat flux for a chopped-cosine distribution, typically F_z вүҲ 1.55 for a fresh core). Both the hot-pin cladding and hot-pin fuel temperatures are tracked with the same first-order thermal inertia model as the average pin.

## **3.5  Coolant-Side Heat Transfer Coefficient**

The coolant-side heat transfer coefficient Оұ is evaluated at each time step by taking the maximum of the forced convection and natural convection values, providing a smooth transition between the two regimes as pump flow decays.

### **3.5.1  Dittus-Boelter Forced Convection**

The Dittus-Boelter [1930] correlation for turbulent forced convection in a rod bundle channel:

Nu=0.023вҖүRe0.8Pr0.4вҖҒ3.11

Оұ=NuвҖүkfDhвҖҒ3.12

where Re = ПҒ v D_h / Ој and Pr = Ој c_p / k are the Reynolds and Prandtl numbers evaluated using subcooled liquid properties at the local pressure and temperature. The laminar limit Nu = 3.66 is applied when Re < 2300. All fluid properties are evaluated at the XSteam library for the local P and T, with temperature clipped at 0.9999 T_sat to ensure subcooled properties are returned.

### **3.5.2  Churchill-Chu Natural Convection**

The Churchill-Chu [1975] correlation for vertical natural convection from a heated surface:

Nu=0.825+0.387Ra1/61+0.492/Pr9/168/272вҖҒ3.13

where Ra = Gr Pr is the Rayleigh number and the thermal expansion coefficient ОІ is evaluated from a cubic polynomial fit to water properties. This correlation is applied simultaneously with Dittus-Boelter and the larger of the two HTC values is used, providing a continuous natural circulation capability as pump speed approaches zero.

### **3.5.3  Film Boiling вҖ” Bromley Correlation**

When the DNBR drops below 1.0, the code switches to the Bromley [1950] pool film boiling correlation for the post-CHF heat transfer coefficient. The implicit form used in FLARE eliminates the requirement to know the wall superheat О”T_sat explicitly:

hfb=CB4/3qavgвҖІвҖІ1/3вҖҒ3.14

where C_B is the Bromley coefficient:

CB=0.62О»v3ПҒvПҒLвҲ’ПҒvghfgОјvDh0.25вҖҒ3.15

Film boiling replaces Dittus-Boelter once activated. Three post-CHF model options are available via the post_chf_model parameter: bromley (default, Bromley throughout post-CHF); cold_leg (conservative ad-hoc model for cold-leg LOCA, described below); and hot_leg (direct Bromley HTC applied immediately on CHF detection). For bromley and hot_leg models, the Bromley HTC is applied directly each time step with no under-relaxation; the HTC cap htc_post_chf defaults to 1Г—10вҒ¶ W/mВІK (effectively uncapped). For the cold_leg model, htc_post_chf is automatically set to 50 W/mВІK (the Leidenfrost floor) and under-relaxation (htc_relax, default 0.3) is applied to produce a physically reasonable gradual transition from the high nucleate-boiling HTC to the 50 W/mВІK film boiling floor, modelling the ad-hoc assumption of immediate flow stagnation and film boiling at the break. Film boiling is activated by two independent triggers: (a) DNBR falling below 1.0, and (b) vessel liquid level dropping below film_boiling_level_m. For the cold_leg model, a third trigger (a2) activates film boiling immediately when the equilibrium quality x_eq вүҘ 0 and the pump has tripped (i.e. at scram time), representing the conservative assumption of immediate DNB from flow reversal in the cold leg. The pump-trip gate on trigger (a2) prevents spurious film boiling activation during the RPS signal processing delay while the pump is still delivering rated flow. Two multiplicative bias parameters allow independent scaling of the core HTC in uncertainty analysis: htc_core_mult (default 1.0) scales the forced-convection HTC before the post-CHF transition; htc_fb_mult (default 1.0) scales all film boiling HTC values after the transition.

## **3.6  Departure from Nucleate Boiling Ratio**

The departure from nucleate boiling ratio (DNBR) is evaluated at each time step when dnbr_flag = 1 and represents the margin between the actual heat flux and the critical heat flux at which nucleate boiling breaks down:

DNBR=qCHFвҖІвҖІqhotвҖІвҖІвҖҒ3.16

where q"_hot = F_r Г— F_z Г— q"_avg is the hot-pin heat flux [W/mВІ]. A DNBR below 1.0 indicates that DNB has occurred and the rod is in film boiling. The minimum DNBR over the transient (MDNBR) is tracked and reported.

### **3.6.1  Biasi / Zuber CHF Correlation**

The critical heat flux is calculated using the Biasi [1967] correlation blended with the modified Zuber [1959] pool boiling CHF model, following the RELAP5/MOD2 and TRACE convention.

The Biasi correlation is expressed in CGS input units (D in cm, G in g/cmВІs, P in bar):

FP=0.7249+0.099PeвҲ’0.032PвҖҒ3.17

HP=вҲ’1.159+0.149PeвҲ’0.019P+9P10+P2вҖҒ3.18

qloвҖІвҖІ=1.5027Г—107Dcm0.6GcgsвҲ’1/6HвҲ’xeqвҖҒ3.19

qhiвҖІвҖІ=2.764Г—107Dcm0.6GcgsвҲ’0.4FвҲ’xeqвҖҒ3.20

and q"_Biasi = max(q"_lo, q"_hi). The Zuber pool boiling CHF is:

The two models are blended linearly between G = 100 kg/mВІs (pure Zuber) and G = 200 kg/mВІs (pure Biasi).

### **3.6.2  Bowring CHF Correlation**

The Bowring [1972] correlation is used as an alternative for the high-pressure regime (P > ~13 MPa) where the Biasi H(P) function approaches zero and the correlation loses validity. The Bowring correlation uses a pressure-normalized form following Tong and Tang [1997]:

n=2.0вҲ’0.5pr,вҖҒвҖҒpr=P6.895вҖ…MPaвҖҒ3.23

where H_bow and C_len are pressure-dependent coefficients. The Bowring correlation is valid in the range P = 7вҖ“17 MPa, G = 200вҖ“6000 kg/mВІs, D = 2вҖ“45 mm.

## **3.7  Cladding Oxidation**

Zircaloy cladding oxidation by steam is calculated using the Baker-Just [1962] parabolic rate law, applicable to the high-temperature steam oxidation regime (T > 1000В°C) relevant to severe LOCA conditions:

dОҙ2dt=AeвҲ’Q/RTcladвҖҒ3.25

where Оҙ is the oxide layer thickness [m], A = 33.3 Г— 10вҒ»Ві mВІ/s is the pre-exponential factor, Q = 45,500 cal/mol is the activation energy, and R is the universal gas constant. The maximum local oxidation fraction (MLO) is evaluated as the ratio of the reacted cladding thickness to the pre-accident cladding wall thickness and is compared against the 10 CFR 50.46 limit of 17%.

# **Chapter 4 вҖ” Reactor Kinetics Model**

Reactor power is calculated using a point kinetics model with six groups of delayed neutrons and net reactivity feedback from fuel temperature (Doppler), moderator temperature, and вҖ” for RIA cases вҖ” an externally prescribed reactivity insertion. The point kinetics model is valid for spatial modes that are stable (fundamental mode dominance), which is the appropriate assumption for the azimuthally symmetric accidents within FLAREвҖҷs scope.

## **4.1  Point Kinetics Equations**

The time-dependent reactor power P(t) and the six delayed neutron precursor concentrations C_i(t) are governed by:

dPdt=ПҒnetвҲ’ОІeffОӣ*P+iвҖӢО»iCiвҖҒ4.1

dCidt=ОІiОӣ*PвҲ’О»iCi,вҖҒвҖҒi=1,вҖҰ,6вҖҒ4.2

where ПҒ_net is the net reactivity [dimensionless], ОІ_eff is the effective delayed neutron fraction, Оӣ* is the prompt neutron generation time, ОІ_i and О»_i are the delayed neutron yield fraction and decay constant for group i. For standard PWR U-235 fuel, the six-group data from the fission_scram() function are used:

| **Group i** | Yield ОІ_i | Decay constant О»_i (sвҒ»В№) |
| --- | --- | --- |
| 1 (shortest) | 0.038 | 2.2138E-01... (22.1) |
| 2 | 0.213 | 5.1587E-01... (0.516) |
| 3 | 0.188 | 1.9594E-01... (0.196) |
| 4 | 0.407 | 1.0314E-01... (0.103) |
| 5 | 0.128 | 3.3656E-02... (0.034) |
| 6 (longest) | 0.026 | 1.1681E-02... (0.012) |

The single-group prompt kinetics approximation used for Euler stepping employs a modified generation time:

Оӣ*=Оӣprompt1вҲ’ПҒnet/ОІeff,вҖҒвҖҒПҒnet<ОІeffвҖҒ4.3

## **4.2  Reactivity Feedback Models**

### **4.2.1  Doppler (Fuel Temperature) Coefficient**

The Doppler reactivity feedback is proportional to the change in average fuel temperature from its initial value:

ПҒD=ОұDTfuelвҲ’Tfuel,0вҖҒвҖҒОұDвҖ…inвҖ…pcm/вҲҳCвҖҒ4.4

where Оұ_D is negative (typically вҲ’2 to вҲ’4 pcm/В°C for PWR conditions), providing prompt negative feedback during power excursions. The Doppler coefficient is specified as an input parameter (alpha_D_pcm) and is applied linearly over the full range of fuel temperatures encountered in the simulation.

### **4.2.2  Moderator Temperature Coefficient**

The moderator temperature coefficient provides feedback from changes in bulk coolant temperature:

ПҒM=ОұMTcoolantвҲ’Tcoolant,0вҖҒвҖҒОұMвҖ…inвҖ…pcm/вҲҳCвҖҒ4.5

where Оұ_M must be negative at power (typically вҲ’20 to вҲ’40 pcm/В°C for PWR full-power conditions) to satisfy the inherent stability requirements of 10 CFR 50. The temperature term is weighted by the cubed liquid fraction (1вҲ’Оұ_void)Ві to capture the physical reduction in moderating effectiveness as the core voids. A separate void reactivity term using the mass steam quality x_eq provides additional negative feedback during LOCA blowdown:

*ПҒ_M = Оұ_M [pcm/В°C] Г— (T_coolant вҲ’ T_coolant,0) Г— (1 вҲ’ Оұ_void)Ві + Оұ_M_void [pcm/%] Г— x_eq Г— 100*

where Оұ_void is the volumetric void fraction computed from x_eq, ПҒ_L, and ПҒ_V at each time step; and Оұ_M_void (alpha_M_void, default вҲ’150 pcm/% steam quality) is the void reactivity coefficient. The (1вҲ’Оұ_void)Ві weighting suppresses the temperature term rapidly as the core voids (at 70% volumetric void the term is reduced to ~2.7%), ensuring that the strong positive temperature feedback from cold coolant during LOCA blowdown does not dominate over the physically correct void-suppression effect. The temperature term correctly captures positive moderator feedback during MSLB cooldown (subcooled вҶ’ positive ПҒ_M), while the void term provides negative feedback during blowdown voiding. The net reactivity in RIA cases is:

ПҒnet=ПҒextt+ПҒD+ПҒMвҖҒ4.6

### **4.2.3  External Reactivity Insertion**

For reactivity-initiated accident (RIA) cases, a time-dependent external reactivity ПҒ_ext(t) [pcm] is read from column F of the time-series input table and evaluated by linear interpolation:

ПҒextt=interpt,ttable,ПҒtableвҖҒ4.7

This table-driven approach allows representation of arbitrary reactivity insertion histories, including rod ejection pulses (step), rod withdrawal (ramp), and boron dilution (slow ramp).

## **4.3  Scram Model**

The Reactor Protection System (RPS) monitors five setpoints continuously when the reactor is at power (full_power_flag = 0 or ria_flag = 1). A trip signal is generated when any setpoint is exceeded:

| **Setpoint** | Default Value | Parameter |
| --- | --- | --- |
| High neutron flux | 110% of rated | trip_power_frac |
| Low RCS pressure | 90% of initial P | trip_P_lo_kPa |
| High RCS pressure | 0 (disabled) | trip_P_hi_kPa |
| Low coolant flow | 0 (disabled) | trip_flow_frac |
| Timed manual trip | 1Г—10вҒ¶ s (never) | trip_time_s |

A signal processing and relay delay (trip_delay, default 1.5 s) is applied between the detection of a trip setpoint and the execution of the scram. This represents the hardware response time of conventional PWR reactor protection systems with hard-wired relays. When pump_trip_time is not specified, the RCP trips coincident with the reactor scram; the pump therefore continues to deliver rated flow throughout the trip_delay window. Film boiling triggers that depend on post-scram flow conditions (specifically the cold_leg Trigger a2, Section 3.5.3) are gated on the _pump_tripped flag to prevent activation at full pump flow during this delay. The total scram rod worth is specified by trip_rod_worth_pcm (default вҲ’15,000 pcm). The scram reactivity insertion follows the six-group delayed neutron decay model implemented in fission_scram(), which takes the actual rod worth in dollars (trip_rod_worth_pcm / ОІ_eff) rather than a generic вҲ’20$ assumption, ensuring that the post-scram fission power decay curve correctly reflects the actual shutdown depth for the case under analysis:

PtrelP0=iвҖӢОІОІвҲ’ПҒОІОІieвҲ’О»itreliвҖӢОІiвҖҒ4.8

where t_rel is the time since scram, ПҒ = вҲ’20 is the scram reactivity insertion in units of ОІ, and the sum is over the six delayed neutron groups. This function produces a rapid initial power drop followed by the characteristic delayed neutron tail.

## **4.4  Prompt Jump Approximation**

For subcritical RIA cases where the reactivity changes faster than the time step resolves, the Prompt Jump Approximation (PJA) is applied to capture the instantaneous prompt-neutron power adjustment:

Pnew=PoldОІeffвҲ’ПҒprevОІeffвҲ’ПҒnetвҖҒ4.9

This approximation is applied only when both ПҒ_prev and ПҒ_net are sub-critical (ПҒ < ОІ_eff) and the time step would otherwise under-resolve the power change. It provides a significant improvement in accuracy for rod ejection simulations without requiring sub-millisecond time steps.

## **4.5  Nordheim-Fuchs Prompt Supercritical Model**

When ПҒ_net exceeds ОІ_eff (prompt supercritical excursion), the standard Euler kinetics solver cannot stably resolve the sub-millisecond power pulse. FLARE applies the Nordheim-Fuchs [1946] analytic model for this regime. The excursion is governed by the coupled adiabatic equations:

dPdt=ОҙПҒвҲ’ОұDE/CfОӣpromptPвҖҒ4.10

dEdt=PвҖҒ4.11

where ОҙПҒ = ПҒ_net - ОІ_eff [pcm] is the initial excess reactivity above prompt critical, E [J] is the energy deposited since the burst began, C_f = ПҒ_fuel c_p,fuel V_fuel is the fuel heat capacity [J/K], and |Оұ_D| is the magnitude of the Doppler coefficient [pcm/В°C]. The burst self-terminates when the adiabatic Doppler feedback (|Оұ_D| E/C_f) consumes all of the initial excess reactivity. FLARE integrates these equations over N_sub = 1000 sub-steps within each master time step, providing accurate resolution of the pulse shape without global time step reduction.

The adiabatic temperature rise accumulated during the burst is added to the Doppler feedback state variable _nf_delta_T, which then decays with the fuel thermal time constant in subsequent steps.

## **4.6  Decay Heat Model**

Following a reactor scram, the fission product decay heat is calculated using the ANS-1979 standard [ANSI/ANS-5.1, 1979]. The decay heat power is expressed as a fraction of the rated thermal power:

PdecaytrelQ0=isotopesвҖӢОұiО»ieвҲ’О»itrel1вҲ’eвҲ’О»iTopвҖҒ4.12

where t_rel is the time since shutdown, T_op is the reactor operating time before shutdown, and the sums are over 23 nuclide groups for U-235, U-238, and Pu-239 fission products. Actinide contributions (U-239 and Np-239) are added separately via dhp_act(). A gamma multiplication factor G(t_rel) accounting for photon transport and capture is applied via tabulated values from the ANS standard.

For non-standard fuel compositions, the decay heat can be overridden by a user-supplied time-series table (column C of the input table, activated by power_table_flag), which is added to the standard correlation result. This supports applications with unusual burnup histories or non-UOвӮӮ fuel types.

# **Chapter 5 вҖ” Rod Failure and Radiological Source Term Model**

## **5.1  Statistical Rod Failure Model**

FLARE tracks cladding failures using a statistical model based on the assumption that the radial power distribution across the N_pins fuel pins in the core follows a Gaussian distribution. The distribution is parametrized by the hot-pin peaking factor F_r and the number of standard deviations k_sigma separating the core-average pin (f = 1.0) from the hot pin (f = F_r):

Пғr=FrвҲ’1kПғвҖҒ5.1

The fraction of pins with power peaking factor above a threshold f_thresh is:

Pf>fthresh=0.5вҖүerfcfthreshвҲ’12ПғrвҖҒ5.2

The default k_sigma = 3.0 places the hot pin at the 99.87th percentile of the distribution. The absolute number of rods failing at each temperature threshold is:

NfailTthresh=NpinsPThot cladf>TthreshвҖҒ5.3

where T_hot_clad(f) is the hot-pin cladding temperature for a pin with peaking factor f. Two failure thresholds are tracked:

- Gap release threshold: T_gap_release = 800В°C вҖ” cladding rupture temperature under internal fission gas pressure

- Early in-vessel threshold: T_early_iv = 1204В°C вҖ” the 10 CFR 50.46 PCT limit

In addition, rods experiencing DNBR < 1.0 are counted as failed for gap release purposes (conservative; assuming DNB leads to cladding failure). When film boiling is active, the rod failure count depends on the trigger: core uncovery (vessel level below film_boiling_level_m) sets N_fail_DNB = N_pins because all rods are physically uncovered; DNBR-only film boiling (no uncovery) applies the Gaussian statistical model with sf(0) = 0.5, representing the statistically hotter half of the radial distribution as the failed population.

## **5.2  RG 1.183 Radiological Source Term**

The radiological source term is estimated using the release fractions specified in NRC Regulatory Guide 1.183 [NRC, 2012], Alternative Radiological Source Terms for Evaluating Design Basis Accidents at Nuclear Power Reactors, Table 3. Release fractions are applied to the number of failed pins in each damage category:

| **Nuclide Group** | Gap Release Fraction | Early In-Vessel Fraction |
| --- | --- | --- |
| Noble gases (Kr, Xe) | 5% | 20% |
| Halogens (I) | 5% | 35% |
| Alkali metals (Cs, Rb) | 5% | 25% |
| Tellurium group | 0% | 5% |
| Barium / Strontium | 0% | 2% |

The total release fraction for each nuclide group is calculated as:

Rgroup=fgap onlyFgap+fearly ivFearly ivвҖҒ5.4

where f_gap_only = (N_gap - N_early_iv) / N_pins is the fraction of rods that experienced gap release only (not early in-vessel), f_early_iv = N_early_iv / N_pins is the early in-vessel failed fraction, and F_gap, F_early_iv are the RG 1.183 release fractions from Table 5-1. These fractions represent the fraction of the total core inventory of each nuclide group released to the RCS.

# **Chapter 6 вҖ” Numerical Solution Methods**

## **6.1  Time Grid and Operator Splitting**

FLARE uses a non-uniform time grid that provides fine resolution near the start of the transient (where pressure and temperature change most rapidly) and coarser steps at later times:

- t = 0 to 0.1 s: О”t = 0.01 s (0.01-s steps, 10 per 0.1 s)

- t = 0.1 to 2 s: О”t = 0.1 s

- t = 2 s to end: О”t = 1 s

The solution advances using an operator-split strategy in which the thermal-hydraulic state (P, h), the kinetics power, and the cladding temperature are solved in sequence within each time step, with one-step lagging at the coupling interfaces.

## **6.2  Thermal-Hydraulic Solver**

The pressure and enthalpy are advanced by solving the 2Г—2 linear system described in Chapter 2. The coefficient matrix A is formed from the partial derivatives of internal energy and specific volume, evaluated numerically using finite differences about the current state:

вҲӮeвҲӮPвүҲeP+О”P,hвҲ’ePвҲ’О”P,h2О”PвҖҒ6.1

The pressure increment is О”P = 10 Pa and the enthalpy increment is О”h = 100 J/kg. The linear solve is performed using numpy.linalg.solve(). If the matrix is singular (which can occur near the critical point where the property derivatives become ill-conditioned), the code falls back to a direct enthalpy update with the pressure held constant.

## **6.3  PORV Predictor-Corrector**

The PORV mass flow rate enters the b vector as a nonlinear term (since G_ERM depends on the current pressure). A predictor-corrector method resolves this coupling: the pressure is first computed without PORV flow (predictor), then with maximum PORV flow. If the pressure remains above the PORV setpoint with maximum flow, the full PORV flow is used. If the pressure falls below the setpoint with maximum flow, the PORV flow fraction that exactly pins the pressure at the setpoint is determined by linear interpolation:

f=PnoneвҲ’PsetpointPnoneвҲ’Pmax PORVвҖҒ6.2

mPORV=fвҖүmPORV,maxвҖҒ6.3

## **6.4  Kinetics Solver**

For sub-critical conditions, the point kinetics equation (4.1) is advanced using a first-order explicit Euler step:

Pt+О”t=Pt1+О”tвҖүПҒnet10вҲ’5Оӣ*вҖҒ6.4

The prompt jump approximation (Section 4.4) is applied before this step to capture instantaneous power adjustments. For prompt-supercritical conditions (ПҒ_net > ОІ_eff), the Nordheim-Fuchs analytic integration (Section 4.5) replaces the Euler step. Post-scram, the total reactor power is the sum of the fission_scram() baseline and an excess fission power term P_excess that captures re-criticalization when ПҒ_net rises above zero due to insufficient rod worth or cold-water moderator feedback. The quasi-static inhour approximation is used for the excess power:

*dP_excess/dt = (ПҒ_net / Оӣ*) Г— P_seed*

where P_seed = max(P_excess, P_base) and P_base = fission_scram(t_rel, ПҒ_$) Г— P_rated is the baseline fission power evaluated at the actual rod worth. This seeding ensures that re-criticalization can develop even after the precursor population has decayed, because the baseline decay curve provides a continuous neutron source. P_excess accumulates when ПҒ_net > 0 (delayed supercritical, growth time constant ПҒ_net/Оӣ*) and decays naturally when ПҒ_net < 0. It is bounded above at rated power and floored at zero.

## **6.5  Heat Conduction Solver**

The cladding slab is discretized with N uniformly-spaced nodes across thickness Оҙ_clad. The implicit time integration produces a tridiagonal system:

aiTiвҲ’1t+1+biTit+1+ciTi+1t+1=diвҖҒ6.5

where the coefficients are:

ai=ci=вҲ’О»cladО”x2вҖҒвҖҒ(internal nodes)вҖҒ6.6

bi=ПҒcladcp,cladО”t+2О»cladО”x2вҖҒ6.7

Boundary conditions enter through the d_i vector. The inner surface heat flux and outer surface convective boundary conditions modify the first and last row of the tridiagonal system. The system is solved efficiently using numpy.linalg.solve() applied to the full NГ—N tridiagonal matrix.

## **6.6  Numerical Stability Considerations**

The semi-implicit pressure solver is stable with respect to the acoustic Courant limit, since sound wave propagation is not resolved in the single-node model. The explicit Euler kinetics step is stable for О”t < Оӣ* / |ПҒ_net| in sub-critical conditions, which is satisfied at 1-s steps for typical scram reactivity values. The Nordheim-Fuchs sub-cycling with N_sub = 1000 steps per master time step ensures accurate resolution of prompt supercritical bursts regardless of the master time step size. The implicit cladding slab solver is unconditionally stable by the von Neumann criterion.

# **Chapter 8 вҖ” Uncertainty Analysis**

## **8.1  Monte Carlo Framework**

The FLARE uncertainty analysis module (flare_ua.py) implements a Monte Carlo sampling loop that drives the simulation engine (flare_sim.py) through N_samples independent runs, each with a different realization of the uncertain input parameters. The sampling uses the numpy random number generator with independent draws for each run and each parameter. Each run is executed as a subprocess and output files are collected in a timestamped run folder (ua_{case}_{YYYYMMDD_HHMMSS}). Previous run folders can be reloaded from the Results tab without re-running the analysis.

## **8.3  Sample Generation**

For each distribution type, samples are drawn as follows:

- Uniform[a, b]: x = a + (b-a) U, where U ~ Uniform(0,1)

- Normal(Ој, Пғ): x = Ој + Пғ Z, where Z ~ Normal(0,1)

- Lognormal(Ој, Пғ): x = exp(Z Пғ + Ој), where Z ~ Normal(0,1)

- Triangular(a, b, mode): x = triangular(a, b, mode) via numpy.random

## **8.4  Output Statistics and 95/95 Criterion**

For each scalar output variable (PCT, MDNBR, peak pressure, etc.), the following statistics are computed over the N_samples runs: mean, standard deviation, minimum, maximum, 5th percentile, 50th percentile (median), and 95th percentile.

The one-sided 95/95 tolerance limit (95th percentile with 95% confidence) applicable under 10 CFR 50.46(b) requires a minimum sample size determined by the Wilks [1941] formula:

NвүҘln1вҲ’ОіlnОІвҖҒвҖҒ[one-sided, order statistic = N]вҖҒ8.1

where Оі = 0.95 is the confidence level and ОІ = 0.95 is the probability content. This gives N вүҘ 59 samples for the one-sided case where the largest value is taken as the 95th percentile estimate. For the two-sided 95/95 case, N вүҘ 93 samples are required.

The Monte Carlo output includes envelope plots (minimum, mean, maximum, 95th percentile) and cumulative distribution functions for each output variable, generated by the Streamlit UA interface.

# **Chapter 9 вҖ” Code Assessment**

## **9.1  Conservation Checks**

The global mass conservation residual is evaluated at each time step and can be monitored in the output. The normalized mass M_scaled = M(t)/M(0) tracks the total inventory. The energy residual is implicit in the 2Г—2 solver: the volume constraint (b[1] = 0 for a sealed system) exactly recovers the pressure change from thermal expansion, providing an internal energy balance check.

## **9.2  Steady-State Initialization**

For rated-power steady-state conditions, FLARE produces the following initial conditions for a 575 MWt PWR/SMR:

- RCS pressure: 15,500 kPa (2250 psia) вҖ” matches typical SMR operating pressure

- Core exit temperature: 312.8В°C вҖ” consistent with 30В°C subcooling at rated pressure

- Rated pump speed: 3550 rpm with homologous curve operating point

- Steam generator heat removal: UA Г— (T_primary - T_sat,secondary)

- Initial cladding temperature: TTwall_ini set from the rated-power D-B HTC and heat flux

## **9.3  Analytical Benchmarks**

The following analytical benchmarks validate individual model components:

### **9.3.1  Nordheim-Fuchs Analytic Solution**

For a step external reactivity insertion ОҙПҒ above prompt critical with initial power P_0, the Nordheim-Fuchs model predicts a peak power:

Ppeak=P0+CfОҙПҒ22ОұDОӣpromptвҖҒ9.1

and a total energy release per unit fuel heat capacity:

О”T=ОҙПҒОұDвҖҒ9.2

The FLARE sub-cycle integration of equations (4.10)вҖ“(4.11) reproduces these analytic results to within 0.5% for О”t = 1 s master steps with N_sub = 1000.

### **9.3.2  Moody Critical Flow Benchmark**

The ERM critical mass flux can be compared against the Moody [1965] charts for steam-water mixtures. For stagnation conditions of P_0 = 6.9 MPa (1000 psia) and x_0 = 0.1, the Moody slip model predicts G_cr вүҲ 8000 kg/mВІs. The FLARE Newton-Raphson solver reproduces this value to within 2% across the PWR pressure range.

### **9.3.3  Baker-Just Oxidation**

The Baker-Just parabolic rate law is integrated exactly by the trapezoidal rule in FLARE. For a constant cladding temperature of 1200В°C for 60 s, the oxidation depth predicted by FLARE matches the Baker-Just analytic integral to within 0.1%.

## **9.4  Limitations and Areas for Future Work**

- Spatial resolution: FLAREвҖҷs single-node approximation cannot resolve axial temperature distributions, counter-current flow, or multi-dimensional effects in the reactor vessel. For applications where these phenomena are important (e.g., boron dilution, sump recirculation), a multi-node analysis is required.

- Two-phase slip: The HEM assumption (S = 1) underestimates the void fraction at low mass flux, leading to potential over-prediction of system pressure during reflood. The Fauske slip model in the break flow calculation partially compensates by correctly representing the phase separation at the break plane.

- Natural circulation: The current model provides a single flow path for natural circulation. Loop geometry, seal/loop-seal clearing, and multi-loop interactions require a multi-node nodalization.

- Containment: FLARE does not currently include a dynamic containment model. The back-pressure at the break is held constant at P_containment (default 101 kPa). A containment energy balance model is planned for a future revision.

# **Chapter 10 вҖ” Source Term and Radiological Dose Models**

Following completion of the thermal-hydraulic and core damage calculation, FLARE computes the radiological source term and performs atmospheric dispersion dose screening using the NOTBADTRAD model. Results are written to the вҖңSource Term,вҖқ вҖңDose,вҖқ and вҖңIodine SpikeвҖқ sheets of the output workbook and are displayed in the Results tab of the PWR Simulator and Risk Assessment tools.

## **10.1  Rod Failure Model (RG 1.183 Source Term)**

FLARE estimates the fraction of fuel rods that experience cladding rupture using a Gaussian distribution model. The average cladding temperature TМ…clad and the hot-pin cladding temperature TвӮҷclad are tracked throughout the transient. The hot-pin temperature is TвӮҷclad = TМ…clad + kПғ В· ПғT, where ПғT is the standard deviation of the pin-to-pin temperature distribution:

ПғT = (Thot вҲ’ TМ…clad) / kПғ  (10.1)

where kПғ (default 3.0) represents the number of standard deviations between the average and hot pin. The cumulative fraction of pins exceeding a failure threshold temperature Tlim is given by the upper tail of the Gaussian:

Nfail = Npins Г— [1 вҲ’ ОҰ(z)]  (10.2)

z = (Tlim вҲ’ TМ…clad) / ПғT  (10.3)

where ОҰ(z) is the standard normal cumulative distribution function. This formulation is applied identically for both LOCA and non-LOCA cases with two temperature thresholds: Tlim = 800В°C triggers gap release, and Tlim = 1204В°C triggers early in-vessel release. DNB does not independently trigger RG 1.183 gap release; fuel damage from DNB is instead captured through the accident-initiated iodine spike model (Section 10.3).

The gap and early in-vessel release fractions are applied to the NUREG-1465 Table 2 core inventory for each nuclide group using RG 1.183 Rev. 0 Table 3 release fractions. The combined source term entering containment for each group is:

Qgroup = Igroup Г— (fgap Г— RGgap + feiv Г— RGeiv)  (10.4)

where Igroup is the NUREG-1465 group inventory scaled to the reactor power rating, fgap and feiv are the peak gap and early in-vessel failure fractions from equations (10.2)вҖ“(10.3), and RGgap and RGeiv are the RG 1.183 Rev. 0 release fractions for each group.

## **10.2  NOTBADTRAD Atmospheric Dispersion and Dose Model**

NOTBADTRAD (Not A Bad Transient Radiological Dose) is FLAREвҖҷs integrated containment leakage and atmospheric dispersion dose screening model. It computes whole-body TEDE (Total Effective Dose Equivalent) at the Exclusion Area Boundary (EAB), Low Population Zone (LPZ), and Control Room (CR) for each nuclide group released to containment.

## **10.2.1  Containment Transport Solver**

For each nuclide group, the containment atmosphere inventory N(t) is governed by a first-order linear ODE with source (release from fuel), leakage to environment, radioactive decay, and spray removal:

dNc/dt = S(t) вҲ’ О»c Nc  (10.5)

where S(t) is the source rate [Ci/s] = Qgroup / release_duration, and the total removal coefficient О»c [sвҒ»В№] is the sum of the leak rate, decay constant (if decay credit is taken), and spray removal coefficient (if deposition credit is taken):

О»c = О»leak + О»decay Г— (credit_decay) + О»rem Г— (credit_deposition)  (10.6)

The default regulatory basis sets credit_decay = False and credit_deposition = False, yielding a conservative no-credit calculation. The ODE is solved numerically using a timestep that is at most half the release duration, ensuring accurate resolution of the source phase. When О»c В· О”t < 10вҒ»вҒ¶, a linear approximation is used to avoid catastrophic cancellation in the exponential formula. The cumulative activity leaked to the environment over the analysis period [0, T] is:

Qleaked = вҲ«вӮҖбөҖ О»leak Nc(t) dt  [Ci]  (10.7)

## **10.2.2  Atmospheric Dispersion (ПҮ/Q)**

Atmospheric dispersion is characterized by the ПҮ/Q factor [s/mВі], the ratio of time-integrated air concentration to total release, evaluated using the Pasquill-Gifford Gaussian plume model for ground-level release:

ПҮ/Q = 1 / (2ПҖ Пғy Пғz u)  (10.8)

where Пғy and Пғz are the horizontal and vertical dispersion coefficients [m] evaluated as power-law functions of downwind distance x for the selected Pasquill-Gifford stability class (default Class F), and u is the mean wind speed [m/s] (default 1.0 m/s). Stability class F with 1 m/s wind speed represents the bounding atmospheric conditions specified in Regulatory Guide 1.145 for EAB and LPZ dose calculations. The dispersion coefficients are capped at 5000 m for Пғz to avoid non-physical values at long distances.

## **10.2.3  Dose Calculation**

The TEDE at the EAB and LPZ receptor locations combines inhalation and cloud-shine (external immersion) contributions:

TEDE = Qleaked Г— (ПҮ/Q) Г— (BR Г— DCFinh + DCFcloud)  (10.9)

where BR is the breathing rate [mВі/s] (default 3.47 Г— 10вҒ»вҒҙ mВі/s per RG 1.23), DCFinh is the inhalation dose conversion factor [rem per CiВ·s/mВі], and DCFcloud is the cloud-shine dose conversion factor [rem per CiВ·s/mВі]. Dose conversion factors are taken from EPA FGR-12 for the nine NUREG-1465 nuclide groups. For the Control Room, the cloud-shine term is excluded (workers are inside the building) and a filter decontamination factor DF is applied to the inhalation term:

TEDECR = Qleaked Г— (ПҮ/Q)CR / DFCR Г— BR Г— DCFinh  (10.10)

The default decontamination factor for halogens is 100 (90% HEPA + 5% charcoal + 5% recirculation credit per NUREG-0570). Noble gases are not filtered (DF = 1). The dose limits applied for screening are: EAB 25 rem TEDE, LPZ 25 rem TEDE, and Control Room 5 rem TEDE, per 10 CFR 50.34 and 10 CFR 50.67.

## **10.3  Iodine Spike Model**

The iodine spike model accounts for the pre-existing coolant activity and its amplification during a reactor transient. It is computed independently of the rod-failure source term and applies to all cases вҖ” LOCA and non-LOCA alike вҖ” whenever a triggering event occurs. Only halogens (iodine and bromine) are released; noble gases and other groups are excluded from this pathway.

## **10.3.1  Equilibrium Spike (500Г—)**

Per Regulatory Guide 1.195 and Standard Review Plan 15.6.5, a reactor trip or PORV opening causes a rapid redistribution of dissolved iodine from the primary coolant to the containment atmosphere. The equilibrium spike inventory is modeled as 500 times the Technical Specification coolant activity limit (default 60 Ојi/g#x03BC;i/g DE I-131) applied to the full primary system mass:

Qequil = 500 Г— acoolant Г— Mprimary  [Ci]  (10.11)

where acoolant is the TS limit coolant activity [Ci/g] and Mprimary is the primary system mass [g]. This spike is triggered whenever a reactor scram or PORV opening is detected.

## **10.3.2  Accident-Initiated Spike (335Г—)**

When fuel damage occurs вҖ” defined as DNB on any rods, or average cladding temperature exceeding the 800В°C gap release threshold вҖ” an additional accident-initiated iodine spike is computed per RG 1.195:

Qaccid = 335 Г— acoolant Г— Mprimary  [Ci]  (10.12)

The two spike contributions are additive. The combined halogen release fraction applied to the NUREG-1465 halogen inventory is:

fISP = min[(Qequil + Qaccid) / IHalogens, 1.0]  (10.13)

The iodine spike dose is computed by a dedicated NOTBADTRAD call using fISP applied to the halogen group only, with all other groups set to zero. The resulting TEDE is reported separately from the rod-failure source term dose in the вҖңIodine SpikeвҖқ output sheet and Results tab display.

## **10.4  Risk Assessment Dose Screening**

The Risk Assessment tool (flare_risk.py) executes each case in the case library as a subprocess and reads the EAB TEDE from the Dose sheet of the output workbook. Cases are plotted on an NEI 18-04 Frequency-Consequence (F-C) chart with event frequency [events/yr] on the y-axis and EAB TEDE [rem] on the x-axis.

Event categories and dose limits follow NEI 18-04 Rev. 1 Table 3-1:

вҖў  Anticipated Operational Occurrence (AOO): frequency вүҘ 10вҒ»ВІ/yr, dose limit 0.05 rem TEDE

вҖў  Design Basis Event (DBE): 10вҒ»вҒҙ вүӨ frequency < 10вҒ»ВІ/yr, dose limit 2.5 rem TEDE

вҖў  Beyond Design Basis Event (BDBE): 5 Г— 10вҒ»вҒ· вүӨ frequency < 10вҒ»вҒҙ/yr, dose limit 25 rem TEDE

Cases with EAB TEDE below the applicable dose limit for their frequency category are classified as PASS. Cases exceeding their limit are classified as EXCEEDS and displayed in red on the F-C chart. Cases with no fuel damage or release are classified as вҖңno releaseвҖқ and plotted at a nominal dose below the left axis. The tool persists frequency assignments between sessions in a CSV file (flare_pra_table.csv) in the FLARE folder.

## **10.5  Noble Gas Dose Conversion Factors**

Noble gases contribute to EAB dose exclusively through cloud-shine (external immersion); inhalation dose is negligible for noble gases. The cloud-shine dose conversion factor for the NUREG-1465 noble gas group in NOTBADTRAD is 2.8 Г— 10вҒ»ВІ rem per CiВ·s/mВі. This value reflects the activity-weighted mean of FGR-12 cloud-shine coefficients for the noble gas mixture at reactor shutdown composition (Xe-133, Xe-135, Kr-88, Kr-85m, and other isotopes), without credit for radioactive decay. The shutdown mix is the appropriate basis for a no-decay-credit conservative calculation because the short-lived high-energy emitters (Kr-88, tВҪ = 2.8 hr; Xe-135, tВҪ = 9.2 hr) are present at their full shutdown activity and contribute disproportionately to the immersion dose.

The full-power equilibrium noble gas mix (dominated by very short-lived Xe-137, Xe-138, and Kr-89) would give a significantly higher activity-weighted DCF (~0.27 rem per CiВ·s/mВі), but these isotopes decay within seconds to minutes of shutdown and are not physically present at the receptor during any credible release scenario. The shutdown-composition DCF of 2.8 Г— 10вҒ»ВІ rem per CiВ·s/mВі is therefore the appropriate conservative basis for post-accident dose screening.

# **Chapter 11 вҖ” AI Event Narrative**

## **11.1  Overview**

The FLARE Simulator Results tab includes an AI Event Narrative feature that generates a structured technical narrative of the completed transient using the Anthropic Claude API. The narrative is produced on demand by clicking the Generate narrative button in the AI Event Narrative expander at the bottom of the Results tab. It is cached in the Streamlit session state for the duration of the session and is regenerated only when the button is clicked again.

## **11.2  Data Summary Passed to the API**

Before calling the API, flare_ui.py constructs a plain-text data summary from the output CSV. The summary includes initial, peak, minimum, and final values with times for the following quantities: RCS pressure; RCS temperature; core power; vessel liquid level; pump mass flow rate; ECCS injection flow (if active); MDNBR and time of minimum; average and hot-pin cladding temperatures; hot-pin fuel temperature; core HTC; steam generator heat removal (if active); equilibrium quality range; and rod failure counts for each damage category. The model is instructed to base its narrative strictly on the provided data and not to invent values.

## **11.3  Narrative Structure**

The system prompt instructs the model to produce a 3вҖ“5 paragraph technical narrative organized as: (1) initiating event and immediate plant response; (2) thermal-hydraulic progression; (3) safety system performance; (4) final plant state and regulatory significance. Standard nuclear engineering terminology is required throughout.

## **11.4  API Configuration**

The API key and model string are read from flare_config.txt in the FLARE working directory. Two entries are recognized:

ANTHROPIC_API_KEY = sk-ant-...

ANTHROPIC_MODEL = claude-sonnet-4-5

The ANTHROPIC_MODEL entry is optional; it defaults to claude-sonnet-4-5 if absent. Updating this entry to a newer model string when Anthropic releases a new version of Claude requires no code changes. The same configuration file is shared by the FLARE Assistant chat in flare_home.py. The narrative feature is disabled and shows a configuration message if the API key entry is absent from flare_config.txt.
